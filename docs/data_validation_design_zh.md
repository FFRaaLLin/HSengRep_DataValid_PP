# 历史业务复刻方案（Excel 入库 + 校验平台）

## 1. 目标
- 把历史 Excel 文件中的收支数据读取到数据库。
- 保留**数据产生日期**（来自 Excel 的`日期`字段）与**系统修改时间**（入库时间/更新时间）。
- 保留每条记录的历史版本（SCD Type 2 思路）。
- 通过 SQL 编写界面配置和执行校验规则（只读权限，不允许修改底表）。
- 支持“本期 Excel 与历史数据库（不含本期）”对比校验。
- 将上传与校验解耦：先上传，再校验。
- 支持上传时即时报错（格式/枚举/账号-类别约束等），并可持续扩展。

## 2. 关于“全部 Excel 入库”还是“提前分表”的建议
结论：**两者都要**，采用“原始全量 + 标准分层”的双轨方案。

1. **原始全量入库（强烈建议）**
   - 每个 Excel 列都落到 `raw` 层（含文件名、行号、导入批次、导入时间）。
   - 价值：可追溯、可重放、规则变更后可回算。

2. **标准分表（同样建议）**
   - 面向查询和校验性能，把核心分析字段整理到标准事实表和维表。
   - 价值：性能稳定、字段语义清晰、规则编写简单。

3. **不要只选其一**
   - 只做全量原始：查询慢、规则难维护。
   - 只做标准分表：丢失原始细节，排障困难。

## 3. 分层架构（建议）
1. **原始层（raw）**：原样落库，尽量不丢字段。
2. **暂存层（stg）**：上传批次待校验数据，承接“上传即校验”。
3. **标准层（dwd）**：字段清洗、类型标准化、字典映射（中文类型转码）。
4. **校验层（dqc）**：规则配置、规则执行结果、异常明细。
5. **展示层（app/report）**：展示异常行、告警摘要、同比环比结果。

## 4. 核心数据模型

### 4.1 原始表 `raw_excel_txn`
- 存储 Excel 原始数据 + 文件元信息 + 行号。
- 保留完整追溯能力。
- 可增加 `raw_payload`（JSON）保底承接未来新增列。

### 4.2 上传暂存表 `stg_excel_txn`
- 用于“上传即校验”。
- 一次上传一个 `batch_id`，先校验，后决定是否入正式层。

### 4.3 标准事实表 `dwd_txn_fact`
关键字段：
- `txn_id`：业务主键（可用文件+行号+关键字段哈希）
- `company_name`
- `currency`
- `biz_date`（对应 Excel 的`日期`）
- `bank_name` / `bank_account`
- `inout_type_code` / `category_l1_code` / `category_l2_code`
- `counterparty_bank` / `counterparty_account`
- `remark`
- `expense_amt` / `income_amt`
- `allocation_match`
- `created_at` / `updated_at`
- `is_current` / `valid_from` / `valid_to`（历史版本）

### 4.4 字典维表（中文转码）
- `dim_inout_type`
- `dim_category_l1`
- `dim_category_l2`

说明：`收支类型/收支大类/收支小类`建议做数字编码（INT）或短码（VARCHAR），分析时关联维表展示中文。

### 4.5 账号-允许类别映射表
- `dim_account_rule_map`
- 用来约束“某账号只能出现某些类型/大类/小类”。

### 4.6 校验规则表 `dqc_rule_def`
- `rule_id`
- `rule_name`
- `rule_type`（格式校验、同比、环比、阈值、白名单、异常增长等）
- `engine_type`（`PROGRAM`/`SQL`）
- `sql_template`（SQL 规则可用）
- `severity`（ERROR/WARN）
- `enabled`

### 4.7 校验结果表
- `dqc_check_run`：一次执行记录（批次、时间范围、执行人、耗时、状态）
- `dqc_check_result`：规则级结果（通过/失败、异常数）
- `dqc_check_detail`：行级异常（定位到 source_file + source_row_num + txn_id）

## 5. “SQL 还是程序”怎么选
建议采用**双引擎**，并分阶段：

1. **上传准入校验（程序优先）**
   - 场景：字段格式、必填、枚举、账号-类别强约束。
   - 要求：上传时立即反馈，阻止脏数据进入正式层。
   - 实现：Python（如 Pandera/Pydantic + 自定义规则）。

2. **分析型校验（SQL 优先）**
   - 场景：同比、环比、异常增长、跨期趋势。
   - 要求：可配置、易审计、可复跑。
   - 实现：规则 SQL + 结果落表。

> 结论：你提到的“某字段格式不能错、某账号必须是某类别”应放在**上传准入层（程序）**；同比环比等放在**SQL 分析层**。

## 6. 上传即报错：推荐流程
1. 用户上传 Excel。
2. 数据进入 `stg_excel_txn`。
3. 触发程序化校验（同步返回 + 异步落库）。
4. 若存在 ERROR 级问题：
   - 阻断入 `dwd`。
   - 返回“错误明细 Excel + 页面表格”。
5. 若仅 WARN 或无异常：
   - 允许入 `raw`/`dwd`。
   - 继续执行 SQL 分析校验。

## 7. 可扩展且不乱掉的规则治理

### 7.1 规则分层
- **L1 Schema 规则**：列名、类型、非空、日期格式。
- **L2 业务规则**：账号-类别、金额符号、调拨匹配。
- **L3 分析规则**：同比、环比、异常增长、波动阈值。

### 7.2 规则元数据化
- 所有规则都在 `dqc_rule_def` 登记（包括程序规则）。
- 字段建议：`rule_version`、`owner`、`effective_from`、`effective_to`。
- 每次执行记录命中规则版本，保证可审计。

### 7.3 发布机制
- 新规则先灰度（仅告警 WARN），观察 1-2 周。
- 稳定后提升为 ERROR 级并开启阻断。

## 8. 关键校验能力（SQL 侧）

### 8.1 上期 vs 本期（环比）
- 按账号、收支类型、收支大类/小类、币种聚合。
- 指标：金额、笔数、均值。
- 规则示例：
  - 本期笔数 > 上期 * 2 且增加绝对值 > 10。
  - 本期金额较上期变化超过 ±50%。

### 8.2 同比
- 与去年同月对比（同维度聚合）。

### 8.3 预设交易类型校验
- 维护“账号-允许交易类型”映射白名单。
- 出现未授权类型直接报错。

### 8.4 特殊交易类型
- 可配置单独规则（如调拨匹配必须成对出现）。

## 9. “本期 Excel 单独校验”场景
场景：本期已入库，但你还想对本期 Excel 再做一次对照。
- 做法：
  1. 上传本期 Excel 到 `stg_current_file`。
  2. 对比历史基线时使用 `biz_date < 本期开始日期` 的数据。
  3. 输出差异报告（新增异常、金额偏差、类型偏差）。

## 10. SQL 编写界面（只读）建议
- 提供“规则 SQL 编辑器”，但执行账号只授予：
  - `SELECT`（事实表、维表、规则视图）
  - 禁止 `INSERT/UPDATE/DELETE/DDL`
- 建议通过后端做 SQL 白名单校验：
  - 仅允许以 `SELECT` 或 `WITH` 开头。
  - 禁止分号串联多语句。
  - 禁止关键字（`drop`, `truncate`, `alter`, `insert`, `update`, `delete`）。

## 11. Python 实现建议
- 技术栈：`pandas + SQLAlchemy + pandera/pydantic + fastapi`。
- 模块拆分：
  1. `ingest`：Excel 上传与标准化。
  2. `precheck`：上传准入校验（程序规则）。
  3. `mapper`：中文类型映射。
  4. `validator_sql`：执行 SQL 规则。
  5. `reporter`：输出报表（Excel/HTML/数据库结果表）。

## 12. 异常展示方式（推荐）
推荐“摘要 + 明细 + 可回写修复建议”：
1. **摘要卡片**：规则名、异常条数、影响金额、严重级别。
2. **明细表格**：文件名、行号、账号、日期、类型、异常原因。
3. **修复建议列**：如“类别应为 X，请检查第 N 行账号映射”。
4. **导出异常 Excel**：便于业务回改后重新上传。

不建议只弹报错文本；应保留行级定位与导出能力。

## 13. 上传与校验是否分开
建议分开两步：
1. **上传入库**（保证数据可追溯、可重跑）。
2. **校验执行**（可重复执行不同规则、不同时间窗口）。

同时在“上传”步骤内增加“准入校验子流程”，兼顾质量与解耦。

## 14. 最小可落地版本（MVP）
- Excel 上传 -> `stg` 暂存。
- 程序准入校验：格式、必填、账号-类别白名单。
- 通过后写入 `raw`/`dwd`。
- SQL 校验：环比 + 同比 + 异常增长。
- 结果表 + 异常明细导出 + 只读 SQL 编辑器。
